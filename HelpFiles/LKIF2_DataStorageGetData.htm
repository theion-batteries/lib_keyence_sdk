Data storage output (LKIF2_DataStorageGetData)
RC WINAPI LKIF2_DataStorageGetData(IN int OutNo,IN int NumOutBuffer,OUT LKIF_FLOATVALUE *OutBuffer,OUT int *NumReceived);
Description Obtain the data stored in the data storage.
Parameter OutNo  Specify the OUT No. of the data to be obtained. The value range is 0 to ((the No. of used OUT)-1).
NumOutBuffer  Specify the number of the data pieces to be obtained within the range of 1 to 1200000.
OutBuffer  Specify the pointer to the measured value structure array.
(It is user's responsibility for reserving the area.)
typedef enum {
LKIF_FLOATRESULT_VALID, // Valid data
LKIF_FLOATRESULT_RANGEOVER_P, // Over range at positive (+) side
LKIF_FLOATRESULT_RANGEOVER_N, // Over range at negative (-) side
LKIF_FLOATRESULT_WAITING, // Wait for comparator result
LKIF_FLOATRESULT_ALARM, // Alarm
LKIF_FLOATRESULT_INVALID, // Invalid (error, etc.)
} LKIF_FLOATRESULT;
typedef struct {
LKIF_FLOATRESULT FloatResult; // Whether the result is valid or not
float Value; //measurement value during LKIF_FLOATRESULT_VALID.
} LKIF_FLOATVALUE;
NumReceived  Specify the pointer to the variable which contains the number of data pieces actually obtained.
Return value RC Refer to "Return code list".
Remark Be careful that this function returns an error while data is being stored in the data storage.
If the number of data pieces stored in the LK-G5000 Series is less than the value of NumOfBuffer, all the stored data will be obtained.
(The content of excessive buffer will not be updated.)
If the number of data pieces stored in the LK-G5000 Series is more than the value of NumOfBuffer, the data as much as the amount of NumOfBuffer will be obtained.
